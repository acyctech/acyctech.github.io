函数的传值参数与传引用参数

1、java中方法参数的传值与传引用

首先看一下参数的类型，我们知道java中的数据类型分两种，一种是基本数据类型，比如数字等字面量，另一种是引用类型，比如对象。

int num = 1;

num中存储的就是1的字面量。

Object obj = new Object();

obj中存储的是什么呢？是obj这个对象的引用，也就是这个对象的地址指针。通过这个指针对这个对象进行访问和操作。

例子不是太好写。

当参数为对象时，如果在内部，对这个对象重新new过，那就和方法外的对象没有关系了。如果参数对象提供了方法修改自身的属性，且在方法内部，没有重新赋引用值，那可以通过调用对象自身的方法进行修改。

知乎上有一个回答非常好。

2、php中函数传值参数与传引用参数

slimlar_text($string1,$string2,$percent),其中$percent可以返回相仿百分比。这个地方也是传引用参数，可以对参数进行修改。

聊聊函数与方法的区别

1.java中的方法与函数

在Java中，方法代表对象的行为，可以分为对象方法（或叫成员方法）和类方法（或叫静态方法）。所谓对象方法，是指方法的调用需要依赖具体的对象；所谓类方法，是指方法的调用不依赖具体的对象，直接使用类名就可完成调用。当然，对象方法和类方法的定义也有区别。

Java中一般不提函数，在c语言中，函数也是用来完成特定的功能或说产生一定的行为，但c中没有对象的概念，它是面向过程的。java中提到函数，指的也是方法。

2.PHP中的方法与函数

在PHPH中，用来完成特定功能的代码块，有三种类型，分别是全局函数、类静态函数和对象方法，或称为函数、类方法和对象方法。函数的调用不依赖具体的对象，类方法的调用也不依赖具体的对象，对象方法的调用需要依赖具体的对象。当然他们的定义也是有区别的。

注意：在php中，连接符使用句点，分量符使用->，构造方法使用__construct，变量前面要加$，变量区分大小写，所以可以使用驼峰命令法，而类名和方法名不区分大小写，所以使用下划线分隔命名法。另外php是弱类型语言，声明属性时不使用类型修饰语。

为什么PHP中引用对象属性或方法的符号是->

作业

在Java、Ruby等面向对象设计语言中，对于引用对象属性或方法的分量符号是句点。

而在PHP中使用的确是->（一个减号加一个大于号）这样一个工作量提高了1倍的方案，那这是为什么呢？

大胆猜测一下，PHP在1994年最初设计的时候，估计还是面向过程的设计方法论占据上风。那个时候，它已经使用了句点做为字符串连接符使用了。所以后来PHP也向面向对象发展的时候，只好用另一种符号来代替了。而单独的符号基本上已经被使用完毕了，那只好使用双符组合，就形成了今天的用法。
